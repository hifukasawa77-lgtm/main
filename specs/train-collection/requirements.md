## 1. 背景・目的

### 1.1 背景

hideのポートフォリオサイト（GitHub Pages）に、子ども向けインタラクティブコンテンツを追加する。市場調査の結果、3〜6歳児を対象とした電車コレクションゲームは「広告ゼロ・課金ゼロ・オフライン動作」という保護者ニーズが強く、競合アプリが収集ループを持たない弱点がある。

### 1.2 目的

- 3〜6歳児が一人でプレイできる直感的な電車収集アプリを提供する
- ガチャ・ミニゲーム・図鑑の三位一体ループでリプレイ性を確保する
- 保護者が安心して子どもに渡せる完全無料・完全オフライン・登録不要のコンテンツにする
- hideポートフォリオのコンテンツ拡充と技術力アピールを兼ねる

### 1.3 ターゲットユーザー

| ユーザー区分 | 詳細 |
|---|---|
| プライマリ | 3〜6歳の子ども（タブレット・スマートフォンでプレイ） |
| セカンダリ | 保護者（横で一緒に見ながら楽しむ / コンテンツ安全性を評価する） |

## 2. 機能要件（MoSCoW法）

### 2.1 Must（必須機能）

#### M-01: ホーム画面
- アプリ起動時に表示される親画面
- 図鑑・ガチャ・ミニゲーム（3種）へのナビゲーションボタンを配置
- 所持コイン数を常時表示
- デイリーログインボーナス（毎日30コイン）を自動付与してモーダル表示

#### M-02: ガチャシステム
- 1回ガチャ: 50コイン消費
- 10回ガチャ: 450コイン消費（50コイン割引）
- ガシャポン筐体のCanvas演出アニメーション（筐体回転 → カプセル排出）
- 入手した車両を画面中央で大きく表示（車両名ひらがな・レア度表示）
- 10回ガチャでSR以上を1枚以上確定するピティシステム
- コイン不足時は操作不可＋「コインがたりないよ」メッセージ表示

#### M-03: 図鑑画面
- 収集済み車両をグリッド表示（1行4両）
- 未収集車両はシルエット表示（「？」表示）
- 収集済み車両をタップ → 車両詳細モーダルを表示
  - 車両名（ひらがな・漢字）・レア度・路線・地域
  - Canvas上で車両が左から右へ走行するアニメーション
- 収集率をプログレスバー表示（XX / 60両）

#### M-04: スタンプラリーミニゲーム
- Canvas描画の日本地図上に新幹線主要30駅をアイコン表示
- 駅タップ → ドアタイミングミニゲーム起動
  - 左右からドアが近づいてくる → 中央ゾーン内でタップ → スタンプ獲得
  - 失敗しても再挑戦可能（失敗ペナルティなし）
- スタンプ獲得でコイン10枚
- 路線コンプリートでコイン100枚 + 称号表示
- 取得済み駅はカラー（グレー→路線カラー）に変化

#### M-05: たんていクイズミニゲーム
- 黒シルエット電車を表示 → 4択ボタンから車両名を選択
- 正解: シルエット→フルカラー変化アニメーション + 走行アニメーション + コイン20枚
- 不正解: 正解車両を表示してコイン5枚（ゲームオーバーなし）
- 難易度選択: かんたん（新幹線のみ）/ ふつう（特急含む）/ むずかしい（ローカル含む）
- 1日5問上限（localStorage で管理、翌日リセット）
- 上限到達時: 「またあしたね」メッセージ表示

#### M-06: たびすごろくミニゲーム
- 日本地図マップ上の36マスをサイコロで進む
- サイコロアニメーション（数字がランダムに切り替わり停止）
- 通常マス: 地域の電車カード1枚 + コイン10〜50
- 特別マス: ドクターイエロー出現 / 季節演出 / ボーナスコイン
- ゴール（36マス目）到達: スペシャル演出 + コイン200 + 周回継続
- ゲームオーバーなし

#### M-07: コインシステム
- 初期付与: 100コイン
- デイリーログイン: 30コイン/日
- ゲーム内獲得でコインを追加
- 課金・広告・外部通信なし（完全ローカル管理）

#### M-08: データ永続化
- すべてのゲームデータをlocalStorageに保存
- 収集済み車両リスト・コイン数・スタンプ・すごろく状態・クイズ記録
- ページリロード後もデータを保持

### 2.2 Should（推奨機能）

#### S-01: サウンド（効果音）
- タップ音・コイン獲得音・ガチャ演出音・正解音・不正解音
- Web Audio API でプロシージャル生成（外部ファイル不要）
- ミュートボタンをホーム画面に設置

#### S-02: アニメーション品質
- 車両走行: requestAnimationFrame による滑らか60fps描画
- ガシャポン回転: イージング（ease-out）付き回転アニメーション
- ボタンタップ: スケールアップ → ダウンのバウンス演出

#### S-03: 称号・実績システム
- 路線コンプリート称号（スタンプラリー連動）
- 全車両コンプリート称号（60両収集完了）
- ホーム画面に取得済み称号を表示

#### S-04: ガチャ演出のバリエーション
- SR以上排出時: 画面全体がフラッシュ + 特別カプセル（金色）演出
- 重複車両入手時: コイン10枚に自動変換

### 2.3 Could（あれば良い機能）

#### C-01: BGM
- Music-Generatorによるプロシージャル生成BGM（ホーム・ゲーム別）

#### C-02: すごろく季節演出
- 現在の月に応じた季節背景（春=桜・夏=海・秋=紅葉・冬=雪）

#### C-03: 車両図鑑の詳細情報
- 実際の運行区間テキスト・最高速度・ひとこと説明

### 2.4 Won't（今回は対象外）

- ユーザー登録・ログイン機能
- オンラインランキング・マルチプレイ
- 課金システム・広告表示
- プッシュ通知
- PWAインストール促進（manifest.jsonは既存流用のみ）
- 車両グラフィックの3D表示

## 3. 非機能要件

### 3.1 パフォーマンス

| 項目 | 目標値 |
|---|---|
| 初回表示（FCP） | 3秒以内（3G回線相当） |
| Canvas アニメーション | 60fps（requestAnimationFrame） |
| localStorage 読み書き | 同期処理で100ms以内 |
| HTMLファイルサイズ | 1ファイル 500KB以内（画像除く） |

### 3.2 アクセシビリティ

| 項目 | 要件 |
|---|---|
| タップ領域 | 最小96px × 96px（子ども向け指サイズ対応） |
| フォントサイズ | 本文最小18px、ボタンラベル最小22px |
| 表示言語 | ひらがな主体（漢字使用時はルビを想定） |
| 色覚 | 赤・緑の区別だけに依存しない色設計 |
| 操作 | タップ／クリック単一操作で完結（ドラッグ不使用） |

### 3.3 オフライン対応

- fetch / XMLHttpRequest 不使用
- trains.json は JavaScript 変数としてインラインに埋め込む（またはsameOriginで読み込む）
- 全アセット（CSS・JS・グラフィック）をhtmlと同一オリジンに配置
- Service Worker は既存 sw.js を流用（新規登録不要）

### 3.4 ブラウザ互換性

| 対象 | バージョン |
|---|---|
| iOS Safari | 15以上 |
| Android Chrome | 100以上 |
| Chrome（デスクトップ） | 100以上 |
| Firefox | 100以上 |
| Canvas API | 必須（非対応環境には「ブラウザを更新してください」表示） |

### 3.5 セキュリティ

- XSS対策: innerHTML にユーザー入力値を渡さない（textContent 使用）
- localStorage のデータはJSON.parse時にtry-catchで例外処理
- 外部URL へのリクエスト禁止（CDNライブラリを除く）
- CDNライブラリはSRI（Subresource Integrity）ハッシュを付与

### 3.6 デザイン制約

- 既存ポートフォリオのスタイル（黒背景 + シアン/パープルアクセント）を継承
- ただし子ども向けコンテンツ内部は明るく親しみやすい色調を使用可
- サイバーパンク的演出（ネオングロウ過多・SF都市風）は禁止

## 4. 制約条件

| 制約 | 内容 |
|---|---|
| 技術スタック | 素HTML / CSS / JavaScript / Canvas API のみ |
| ライブラリ追加 | CDN経由のみ。ビルドツール不使用 |
| ファイル配置 | `train-zukan.html` + `assets/trains/` + `data/trains.js` |
| 商標 | 車両の愛称名は使用可。特定企業のロゴ・キャラクターIPは使用禁止 |
| 課金 | 完全無料。ゲーム内コインはリアルマネーと無関係 |
| 外部通信 | 禁止（オフライン完全動作） |
| ゲームオーバー | いかなる状況でも子どもが詰まらない設計（ゲームオーバーなし） |
