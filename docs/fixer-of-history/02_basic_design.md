# 影の権力者（Fixer of History）基本設計書

## 1. 全体方針

「影の権力者」は、既存ポートフォリオ内のブラウザゲームと同じく、単体HTMLで完結するゲームとして実装する。

ゲームロジック、描画、UI制御は `fixer-of-history.html` 内の素のJavaScriptで管理する。Canvas APIを中心に地図ボード、拠点、街道、偉人コマを描画し、周辺UIはHTML/CSSで構成する。

## 4. デザイン方針

### 4.1 トーン

- 黒背景を基調にした歴史ミステリー調
- シアン / パープル系をアクセントとして控えめに使用
- Glassmorphismカード
- 和紙、墨、金箔、朱印を連想させる質感を軽く入れる
- 薄暗い書斎で密書を扱う緊張感

### 4.2 禁止表現

- サイバーパンクテーマ
- SF都市風の背景
- 過度なネオングロウ
- 画面全体が単一の紫や青だけに見える配色

### 4.3 画像アセット

ユーザー提示画像 `watermarked_img_3306347469789567775.png` は雰囲気参照として扱う。

実装時に画像ファイルがリポジトリへ提供される場合:

- 配置先候補: `assets/images/fixer-of-history-hero.png`
- トップページカードまたはゲームページ導入ビジュアルに使用

画像ファイルがない場合:

- CSSとCanvasで暗い書斎風の抽象背景、密書、地図、駒を表現する

## 6. ターン設計

### 6.1 フェイズ

```text
planning -> upheaval -> harvest -> cleanup -> nextTurn
```

### 6.2 策謀フェイズ

- 人間プレイヤーのアクションを2回受け付ける
- CPUフィクサーのアクションを順番に自動解決する
- すべてのアクションが終わると激動フェイズへ進む

### 6.3 激動フェイズ

- 偉人の行動順を決める
- 移動または攻撃を行う
- 同じ拠点に複数偉人がいる場合は合戦処理を実行する

### 6.4 収穫フェイズ

- 拠点支配と恩義に応じて資金配当を行う
- 配当結果をログに出す

### 6.5 ターン終了処理

- パラドックスイベント判定
- 悪名12以上の粛清判定
- 手札補充
- 10ターン終了なら結果画面へ

## 7. CPU設計

CPUは完全最適化ではなく、分かりやすい擬似戦略で動かす。

### 7.1 CPU投資判断

優先度:

1. 生存している
2. 資金が集まっている
3. 拠点価値が高い場所に近い
4. 自分の恩義が少なすぎない

### 7.2 CPU密書判断

- 自分が恩義を持つ偉人を価値の高い拠点へ誘導する
- 暫定1位の支援偉人に攪乱を使う
- 自分の悪名が高い場合は隠蔽を使う

### 7.3 CPU流言判断

- 暫定1位を優先して告発する
- 対象が特定偉人に3以上の恩義を持つと推定される場合に実行する

### 7.4 CPU乗り換え判断

- 敗者への恩義が3以上
- 勝者が生存有力偉人
- CPUの悪名が9未満

上記を満たす場合、一定確率で乗り換える。

## 8. 得点設計

最終得点:

```text
影響力 = 生存偉人への恩義合計
       + 支援偉人が占拠する拠点価値ボーナス
       - 悪名ペナルティ
```

悪名ペナルティ:

```text
0-3: 0
4-6: -3
7-9: -7
10-11: -12
12以上: 粛清により勝利判定対象外
```

## 9. ローカライズ設計

ゲームページ内で `lang` 状態を持ち、日本語と英語を切り替える。

対象:

- タイトル
- ボタン
- アクション名
- カード名
- ログ定型文
- 結果表示

保存:

- MVPでは `localStorage` に言語設定を保存してもよい
- 保存しない場合もゲーム進行に影響しない

## 10. アクセシビリティ

- ボタンには明確なテキストを付与する
- Canvas上の状態はステータスパネルにもテキスト表示する
- 色だけで状態を判別させない
- スマートフォンでタップ可能なサイズを確保する

## 11. リスクと対応

### 11.1 ルールが複雑になりすぎる

対応:

- MVPではアクションを4種類に絞る
- 密書カードを5種類に絞る
- ログで自動処理の理由を短く表示する

### 11.2 偉人AIが読みにくい

対応:

- 偉人一覧に「次の狙い」を表示する
- 移動理由をログに出す

### 11.3 スマホで情報量が多い

対応:

- ボード、アクション、情報を縦積みにする
- 重要な操作をアクションパネルに集約する

### 11.4 既存サイトの雰囲気から浮く

対応:

- トップページカードは既存 `.work-card` に合わせる
- ゲームページは黒背景 + 控えめなアクセントで統一する
