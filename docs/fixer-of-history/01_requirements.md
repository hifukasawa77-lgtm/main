# 影の権力者（Fixer of History）要件定義書

## 1. 目的

ポートフォリオサイトのゲーム一覧に、戦国時代を題材にしたボードゲーム風ブラウザゲーム「影の権力者（Fixer of History）」を追加する。

プレイヤーは表舞台の武将ではなく、裏から歴史を操るフィクサーとして、偉人への投資、密書、告発、乗り換えを使い、最終的な影響力を最大化する。

## 2. 対象ユーザー

- ポートフォリオ訪問者
- ブラウザで短時間遊べる戦略ゲームを求めるユーザー
- 将棋、花札、囲碁など既存のボードゲーム系コンテンツに興味があるユーザー
- 歴史シミュレーションや交渉ゲームが好きなユーザー

## 3. スコープ

### 3.1 MVPで実装する範囲

- 新規ゲームページ `fixer-of-history.html` の追加
- `index.html` の Works / Games セクションへのゲームカード追加
- Canvas API による地図ボード描画
- 1人プレイ + CPUフィクサー 3人
- 偉人コマの自動行動
- 10ターン制
- 投資、密書、流言飛語、主人の乗り換え
- 合戦、死亡、拠点占拠、収穫
- 悪名、粛清、パラドックスイベント
- 日英バイリンガル表示の基本対応
- レスポンシブ対応
- GitHub Pages でそのまま動作する静的HTML構成

### 3.2 MVPでは扱わない範囲

- オンラインマルチプレイ
- アカウント、セーブデータのクラウド保存
- 外部API連携
- ビルドツール導入
- 複雑な史実シナリオ分岐
- 商用ボードゲーム並みの完全な物理コンポーネント再現

## 4. ゲーム体験要件

### 4.1 体験コンセプト

薄暗い書斎で密書を回し、表舞台の英雄たちを静かに動かす緊張感を出す。

ただし、既存リポジトリ方針に従い、サイバーパンク演出、過度なネオングロウ、SF都市風の装飾は使わない。

### 4.2 プレイヤー目標

10ターン終了時に、以下を満たす高得点を目指す。

- 生存している有力な偉人から多くの恩義を得ている
- 悪名を低く抑えている
- 支援した偉人が価値の高い拠点を占拠している
- 粛清されていない

### 4.3 プレイ時間

- 1プレイ: 10分から20分程度
- 1ターン: 1分前後

### 4.4 難易度

MVPでは難易度を固定する。

将来的には以下の難易度追加を想定する。

- Easy: CPUの告発精度が低い
- Normal: 標準
- Hard: CPUが勝ち馬を読み、悪名管理も行う

## 5. 機能要件

### 5.1 トップページ追加

`index.html` の Works / Games セクションにカードを追加する。

- カテゴリ: `board` または `sim`
- タイトル: `影の権力者`
- 英語タイトル: `Fixer of History`
- リンク先: `fixer-of-history.html`
- 説明文: 裏から偉人を操る戦略ボードゲームであることを短く説明
- 既存カードと同じ `.work-card` 構造を使う

### 5.2 ゲームページ

`fixer-of-history.html` は単体で動作する。

- HTML / CSS / JavaScript を同一ファイル内に収める
- 外部ライブラリは使わない
- Canvas API でボードを描画する
- PCとスマートフォンの両方で操作できる

### 5.3 ゲーム開始

- プレイヤー名は固定で `あなた / You`
- CPUフィクサーを3人生成する
- 偉人を初期拠点に配置する
- 各フィクサーに初期資金と初期密書カードを配る
- 10ターンの1ターン目から開始する

### 5.4 ターン進行

各ターンは以下のフェイズで進行する。

1. 策謀フェイズ
2. 激動フェイズ
3. 収穫フェイズ
4. ターン終了処理

### 5.5 策謀フェイズ

人間プレイヤーは1ターンに2アクション行う。

選択可能アクション:

- 投資
- 密書の送付
- 流言飛語
- 休息

CPUフィクサーも同じく2アクションを自動実行する。

### 5.6 投資

- 任意の生存偉人に1から3金を投資できる
- 投資額と同じだけ恩義を得る
- 投資した資金はその偉人の行動や合戦に影響する
- 所持資金を超える投資はできない

### 5.7 密書の送付

手札の密書カードを1枚使用する。

MVPの密書カード:

- 進軍命令: 指定した偉人を隣接拠点へ移動させる
- 密命襲撃: 指定した偉人に同拠点または隣接拠点の敵を狙わせる
- 賄賂: 指定した偉人への自分の恩義を+2する
- 攪乱: 指定した偉人の次の合戦値を-2する
- 隠蔽: 自分の悪名を-1する

### 5.8 流言飛語

- 対象プレイヤーと対象偉人を指定して告発する
- 対象プレイヤーがその偉人に一定以上の恩義を持つ場合、対象の悪名を上げる
- 告発が外れた場合、告発者の悪名が上がる

### 5.9 激動フェイズ

偉人は自動処理で行動する。

- 資金が最も多い偉人から順に行動する
- 各偉人は野望値、現在拠点、周辺拠点価値、隣接敵の有無に基づいて移動または攻撃する
- 同じ拠点に複数偉人がいる場合、合戦を解決する

### 5.10 合戦

合戦値は以下で計算する。

```text
合戦値 = 1D6 + 偉人の野望値 + その偉人に置かれた総資金 + 密書補正
```

高い側が勝利する。

- 敗者は死亡し、ボードから除外される
- 敗者に紐づく資金は失われる
- 敗者への恩義は最終得点対象外になる
- 同値の場合は野望値が高い偉人が勝つ。それも同値なら攻撃側が勝つ

### 5.11 主人の乗り換え

合戦で支援偉人が敗北したとき、人間プレイヤーは乗り換えを選択できる。

- 敗者への自分の恩義を半分切り捨てで勝者へ移す
- 自分の悪名を+3する
- 敗者への投資資金は失われる
- CPUは一定確率で乗り換える

### 5.12 収穫フェイズ

生存偉人が拠点を占拠している場合、その偉人に恩義を持つプレイヤーへ配当が発生する。

```text
配当 = floor(拠点価値 * 自分の恩義 / その偉人への全プレイヤー恩義合計)
```

最低配当は0金。

### 5.13 悪名と粛清

- 悪名は0以上で管理する
- 悪名12以上のプレイヤーは粛清され、勝利判定から除外される
- 悪名は乗り換え、外れ告発、過剰な癒着で上昇する
- 隠蔽カードなどで軽減できる

### 5.14 パラドックスイベント

特定プレイヤーが勝ちすぎている場合、イベント発生率が上がる。

発生条件:

- 暫定1位の影響力が2位より5点以上高い
- または特定偉人への恩義が1人に偏っている

MVPイベント:

- 一揆: 最も価値が高い占拠拠点の支配を解除
- 病死: 最も資金が集まった偉人に死亡判定
- 露見: 暫定1位の悪名+2
- 反動: 暫定1位が持つ密書を1枚失う

## 6. 非機能要件

- ビルド不要
- GitHub Pagesで静的配信できる
- 主要ブラウザの最新版で動作する
- 画面幅360px程度のスマートフォンでも破綻しない
- Canvasが非対応の場合は簡易メッセージを表示する
- 既存サイトのデザインを壊さない
- XSSにつながるHTML挿入を避ける
- ユーザー入力をHTMLとして直接描画しない

## 7. 受け入れ条件

- `index.html` のゲーム一覧から `fixer-of-history.html` へ遷移できる
- 1ゲームを開始し、10ターン終了まで進行できる
- 投資、密書、流言飛語、乗り換えがUIから実行できる
- 合戦で偉人が死亡し、得点計算に反映される
- 悪名12以上で粛清される
- 最終結果画面に順位、影響力、悪名、勝者が表示される
- PCとスマートフォンで主要UIが操作できる

## 8. 承認後の作業

本書、基本設計書、詳細設計書の承認後、実装フェーズへ進む。

実装時は Generator 相当として以下を行う。

- `fixer-of-history.html` 作成
- `index.html` へのカード追加
- 必要に応じた `sitemap.xml` 更新
- ブラウザ表示確認
- Evaluator 相当の仕様適合チェック
