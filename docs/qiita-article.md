## 技術スタック

**フレームワーク: なし**

すべて素の HTML / CSS / JavaScript で実装しています。ビルドツールなし、パッケージマネージャーなし。

```
- HTML5
- CSS3（Flexbox / Grid / CSS Variables）
- JavaScript ES6+（クラス、モジュールパターン）
- Canvas API（ゲームグラフィック・アニメーション）
- Web Audio API（効果音）
- LocalStorage（スコア・設定の保存）
```

一部ゲーム（グラジウス）はReact + BabelをCDN経由で使用。

## Claude AIとの協働方法

### エージェントパイプライン設計

プロジェクト後半では、Claude Codeのマルチエージェント機能を活用した開発パイプラインを構築しました。

```
要件定義（Planner）
  ↓
グラフィック制作（Designer）
  ↓
実装（Generator）
  ↓
テスト（Tester）
  ↓
品質評価（Evaluator）→ 合格でGitHubプッシュ
```

Plannerに要件を伝えると、仕様書を自動生成し、Generator→Tester→Evaluatorの順に自動で動いて最終成果物を提出してくれます。

### 実際のプロンプト例

**将棋AIの場合:**
```
「藤井颯太と羽生善治のAIと対局できる将棋ゲームを作って。
難易度は3段階。詰め将棋のヒント機能あり。
デザインはブラック系で落ち着いた雰囲気に。」
```

Claude AIが盤面描画（Canvas API）、AI思考（ミニマックス法）、UIを全部実装してくれます。

### Claude AIが特に得意なこと

1. **ゲームロジックの実装**: ルール判定、AI思考アルゴリズム、当たり判定
2. **Canvas API描画**: 盤面・スプライト・アニメーション
3. **UIデザイン**: CSSアニメーション、グラスモーフィズムなどのモダンスタイル
4. **デバッグ**: エラー原因の特定と修正

## 実装で苦労したポイント3選

### 1. 麻雀の役判定

麻雀は役の種類が多く（50種以上）、ロン・ツモ・リーチ・鳴き・槓など状態管理が複雑です。

Claude AIに「完全な役判定ロジックを実装して」と依頼したところ、数百行の判定コードを生成してくれましたが、最初のバージョンでは「国士無双」や「七対子」のエッジケースでバグがありました。

**解決策**: テストケースを細かく指定し、「このような手牌の場合、正しい役判定をしてください」と具体例を示すことで精度が上がりました。

### 2. Canvas APIの座標系とHiDPI対応

ゲームをRetinaディスプレイで表示すると、描画がぼやける問題が発生しました。

```javascript
// HiDPI対応
const dpr = window.devicePixelRatio || 1;
canvas.width = width * dpr;
canvas.height = height * dpr;
canvas.style.width = width + 'px';
canvas.style.height = height + 'px';
ctx.scale(dpr, dpr);
```

Claude AIにこの問題を伝えると即座に対応コードを提示してくれました。

### 3. AIの思考速度とUI応答性

チェス・将棋などの思考系ゲームでは、AIの計算中にUIが固まる問題がありました。

**解決策**: Web WorkerかsetTimeout分割でメインスレッドをブロックしないようにしました。Claude AIが`requestAnimationFrame`との組み合わせ実装を提案してくれました。

## まとめ

Claude AIとのゲーム開発で気づいたこと:

- **Claude AIは「実装してくれるコーダー」ではなく「設計も一緒に考えるパートナー」**
- 要件を曖昧に伝えるより、具体的な挙動・デザイン・制約を明示するほど品質が上がる
- バグ修正は「このエラーが出る」と具体的に伝えると解決が早い
- 複雑な機能は小さなステップに分けて依頼する
