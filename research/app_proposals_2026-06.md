# アプリ提案書: 売上を伸ばせるアプリ4案（2026年6月）

- 提案日: 2026-06-10
- 提案先: PM（深澤）
- 根拠資料: [market_research_2026-06.md](./market_research_2026-06.md)（Researcher×3の市場調査）
- 前提: 個人開発 / Web技術＋ラッパー / 収益化は広告＋買い切り / 有料APIキー不使用

## 全体結論

- **プラットフォーム**: Android（Capacitor＋AdMob）を主戦場、Windows（Tauri/Steam＋買い切り）を併売。単一選択ならAndroid
- **収益モデル**: 「無料＋広告＋買い切りで広告除去/機能解放」のハイブリッドが定石
- **推奨順**: 提案1を先行リリースし、そのDAU/継続率データを見て提案2へ本格投資する二段構え

## 提案1（最優先・先行）: デイリー詰将棋「1日1問」習慣化アプリ — Android

**コンセプト**: Wordle型の「1日1問＋ストリーク＋結果シェア」詰将棋。既存 `shogi.html` の盤面・駒ロジック資産を流用。

| 項目 | 内容 |
|---|---|
| プラットフォーム | Android（Capacitor＋AdMob）。Web版/itch.ioで先行需要検証 |
| 収益化 | リワード広告（ヒント解放・もう1問）＋インタースティシャル＋買い切り広告除去 |
| 市場根拠 | 将棋ウォーズ800万人が示す国内将棋人口。既存詰将棋アプリは問題集型で習慣化設計が空白。日本のリワードeCPM $15〜30は世界最高水準 |
| 推定収益 | DAU500〜3,000で月1〜10万円（個人事例レンジと整合） |
| 開発規模 | 小（数週間で出荷可能）。失敗コスト最小 |
| 主なリスク | 詰将棋問題の著作権 → **問題は自動生成とし、Legal-Checker必須**。ASO（ストア検索最適化）が成否を握る |

## 提案3（差別化オプション）: 将棋AI日本語コーチ — Android/Windows両対応

**コンセプト**: WASM将棋エンジン（やねうら王系・ブラウザ動作実績あり）＋オンデバイス処理で「なぜその手が悪いか」を初級者向け日本語で解説する学習特化アプリ。

| 項目 | 内容 |
|---|---|
| プラットフォーム | Android（Capacitor＋広告）＋Windows（Tauri＋買い切り）の両取り |
| 市場根拠 | 評価値表示アプリは飽和する一方「日本語解説」は大手（銀星将棋10）がまだ開発中の空白地帯。個人開発オンデバイスAI将棋アプリ（きふみAI・将棋Lab）の受容実績あり。サーバーコストゼロでAPIキー禁止ルールと整合 |
| 開発規模 | 中〜大（実装難度Medium〜High） |
| 主なリスク | 解説品質の作り込み。対応端末フォールバック設計とモデル初回DLのUXが必須要件 |

## 提案4（ゲーム外の保険）: 業種特化の現場計算ツール — Android

**コンセプト**: 建設・電気・設備など特定職種の「現場で毎日使う計算」を1タップ化（勾配計算、電線選定等）。日本の規格・単位に準拠。

| 項目 | 内容 |
|---|---|
| プラットフォーム | Android（Capacitor）。画面内完結ツールなのでWeb技術と相性良 |
| 収益化 | 無料＋広告＋買い切りで広告除去/機能解放 |
| 市場根拠 | 業種特化は「価格感度が低く最も収益性の高いジャンルの一つ」との分析。日本規格準拠のものは少なく検索流入型で競合薄 |
| 推定収益 | 月0.5〜5万円/本。複数本展開で月3〜15万円 |
| 開発規模 | 小〜中 |
| 主なリスク | 業務知識の調達コスト。計算誤りが実害につながるため免責設計必須 |

## 共通の実行上の注意

1. **Google Play個人アカウントは「12テスター×14日」のクローズドテスト必須** — リリース計画に必ず織り込む（Microsoft Store/itch.ioにこの壁はない）
2. **TWA不可・Capacitor一択**（Android×広告の場合） — アセット完全同梱・オフライン動作で「薄いラッパー」却下を回避。API 35ターゲット
3. **itch.io/Webで即日公開→需要検証→ストア展開**の順序でリスク最小化
4. 著作権・素材ライセンスは各案ともLegal-Checkerのチェックを必須工程とする
