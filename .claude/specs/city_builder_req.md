# 要件定義書 — City Builder ブラッシュアップ

**作成日:** 2026-05-17
**対象ファイル:** `city.html`
**ホスティング:** GitHub Pages（hide_0001_hon リポジトリ）

---

## 1. 背景・目的

個人ポートフォリオに収録済みの City Builder ゲームを大幅拡張する。
現状は道路・住宅・商業・工業の4種類しかなく、インフラシステムや交通アニメーション、
災害イベントが存在しない。SimCity ライクな都市経営体験を Canvas API のみで実現し、
ポートフォリオとしての完成度・プレイアビリティを高める。

現在の実装課題:
- テーマが「CYBER CITY BUILDER」(サイバーパンク) となっており CLAUDE.md の禁止規約に違反
- 施設が 4 種 (ROAD / RES / COM / IND) のみで戦略性が薄い
- インフラ接続・交通アニメーション・災害イベントが未実装

## 2. 機能要件一覧（MoSCoW法）

### Must（必須）

| ID | 機能 | 概要 |
|----|------|------|
| M-01 | テーマ変更 | タイトル・色調をサイバーパンクから「モダン都市計画」スタイルに変更。黒背景+シアン/パープル系アクセントは継承 |
| M-02 | 道路ネットワーク | 既存の道路タイルを継承。住宅が道路に隣接すると車アニメーションが生成される |
| M-03 | 線路・駅 | 線路タイル＋駅タイルを追加。線路が接続された駅間を電車が往復アニメーション |
| M-04 | 電力システム | 発電所タイルを設置し、送電線タイルで各施設に接続しないと施設が機能しない |
| M-05 | 水道システム | 水道管タイルを設置し、各施設に接続しないと施設が機能しない |
| M-06 | 犯罪・事故システム | 人口増加に比例して犯罪発生率が上昇。警察署で軽減。未対策時は人口が徐々に減少 |
| M-07 | 火災システム | ランダムで火災発生（商業施設はリスク高）。消防署で鎮火。被災建物はダメージ状態へ |
| M-08 | 公害システム | 工業施設が多すぎると公害値上昇→幸福度低下→人口減少（既存機能を強化） |
| M-09 | 公園 | 公園タイルを追加。隣接住宅の幸福度ボーナス。人口増加効果あり |
| M-10 | 地震イベント | ランダム発生。施設にダメージ、道路が破損状態になる |
| M-11 | 施設ステータス表示 | タイル右クリックまたはホバーで施設の詳細（稼働状態・電力・水道接続状況）をポップアップ表示 |
| M-12 | UIリニューアル | ツールバーをタブ式に拡張（基本・交通・インフラ・公共施設） |

### Should（推奨）

| ID | 機能 | 概要 |
|----|------|------|
| S-01 | 空港 | 空港タイルを追加。設置後、飛行機が画面を横断するアニメーション |
| S-02 | ガス管システム | ガス管タイルで各施設に接続。未接続時は工業・商業の効率が下がる |
| S-03 | 台風イベント | ランダム発生。施設ダメージ。アニメーション演出あり |
| S-04 | 原子力発電所 | 高出力発電所。確率でメルトダウン発生→広域で人口激減・施設破壊 |
| S-05 | ミニマップ | Canvas右下にミニマップを常時表示 |
| S-06 | グリッド拡張 | グリッドを 30×24 タイルに拡張し、スクロール対応 |

### Could（任意）

| ID | 機能 | 概要 |
|----|------|------|
| C-01 | 季節変化 | 春夏秋冬で背景色・木のグラフィックが変化 |
| C-02 | 統計グラフ | 人口・収入の推移をCanvas内グラフで表示 |
| C-03 | セーブ/ロード | localStorage を使ったゲーム状態の保存・復元 |

### Won't（今回スコープ外）

| ID | 機能 | 理由 |
|----|------|------|
| W-01 | マルチプレイヤー | ビルドツール不使用のため WebSocket 等の導入が困難 |
| W-02 | 3D描画 | Canvas API 2D のみの方針 |
| W-03 | BGM・SE | ポートフォリオ用途のため音声は省略 |

## 3. 非機能要件

| 項目 | 要件 |
|------|------|
| パフォーマンス | 60fps を維持。施設数が最大（20×16=320タイル全埋め）でも描画落ちしないこと |
| セキュリティ | XSS 脆弱性ゼロ。innerHTML へのユーザー入力注入禁止 |
| 互換性 | Chrome / Edge / Firefox 最新版。モバイル（タッチ操作）でも基本操作可能 |
| ファイルサイズ | city.html 単体で完結。外部 JS/CSS ファイル分割なし。1MB 以内 |
| アクセシビリティ | 日英バイリンガル表記（UI ラベルは日本語メイン、英語サブ） |

## 4. 制約条件

- フレームワーク不使用（素の HTML / CSS / JavaScript）
- Canvas API のみで描画（WebGL 不使用）
- CDN 経由ライブラリは許可するが今回は不要
- ビルドツール不使用（Vite / Webpack 等なし）
- **禁止**: サイバーパンクテーマ（ネオングロウ過多、SF都市風演出など）
- カラースキーム: 黒背景 `#000010` + シアン `#00cccc` / パープル `#8844cc` 系アクセント
- グラスモーフィズムはUI要素（ポップアップ等）に適用可
- 初期資金: $10,000（現行 $5,000 から増額し、インフラ設置コストを考慮）
- 現行の20×16グリッドは維持（S-06スクロール拡張は Should 扱い）

## 5. ゲームバランス方針

- 電力・水道を未接続のまま施設を置いても機能しない → 序盤からインフラ整備が必要
- 人口増加に伴いリスク（犯罪・火災・事故）も増加 → 公共施設が必要になる
- 原子力はハイリスク・ハイリターン（Should 扱い）
- ゲームオーバー条件: 資金がマイナスになる
- ゲームクリア条件: 人口 1,000 人達成（現行 500 から変更）
