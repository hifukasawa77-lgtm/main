## 3. データ構造の概要

### 3-1. グリッドセル（Cell オブジェクト）

```
cell = {
  type: number,        // タイルタイプ定数 T.XXXX
  xp: number,          // 施設レベルアップ用経験値
  pol: number,         // 公害値（工業から拡散）
  hp: number,          // 耐久値（100=正常、0=破壊）
  powered: boolean,    // 電力接続フラグ
  watered: boolean,    // 水道接続フラグ
  gassed: boolean,     // ガス接続フラグ（Should）
  fireTimer: number,   // 火災残存時間（0=正常）
}
```

### 3-2. ゲーム統計（stats オブジェクト）

```
stats = {
  money: number,       // 現在資金
  pop: number,         // 人口
  happiness: number,   // 幸福度 0-100
  year: number,        // 経過年
  crimeRate: number,   // 犯罪発生率 0-100
  fireRisk: number,    // 火災リスク 0-100
  pollution: number,   // 総公害値
}
```

### 3-3. 乗り物（Vehicle オブジェクト）

```
vehicle = {
  type: 'car' | 'train' | 'plane',
  x: number,           // Canvas 上の X 座標（px）
  y: number,           // Canvas 上の Y 座標（px）
  dx: number,          // X方向速度（px/frame）
  dy: number,          // Y方向速度（px/frame）
  path: [[r,c], ...],  // 経路（車・電車のみ）
  pathIdx: number,     // 現在の経路インデックス
  progress: number,    // タイル内移動進捗 0.0-1.0
  color: string,       // 描画色
  active: boolean,     // 有効フラグ
}
```

### 3-4. イベントキュー（event オブジェクト）

```
event = {
  type: 'fire' | 'quake' | 'typhoon' | 'meltdown',
  r: number,           // 発生行
  c: number,           // 発生列
  severity: number,    // 深刻度 1-3
  timer: number,       // 残存フレーム数
}
```

### 3-5. インフラネットワーク

```
// Union-Find を使ったネットワーク管理
network = {
  power: UnionFind,    // 発電所と送電線と施設の接続グループ
  water: UnionFind,    // 水源（常時供給）と水道管と施設の接続グループ
  gas: UnionFind,      // ガス源と配管と施設の接続グループ（Should）
}
```

## 4. 主要コンポーネントの役割

### 4-1. Constants（定数モジュール）

タイルタイプ定数 `T`、各施設の初期コスト・維持費・効果値・制約を一元管理する。

### 4-2. State（状態管理）

`grid`、`stats`、`vehicles`、`events`、`particles` を一箇所で宣言・初期化する。
`init()` で全ステートをリセット。

### 4-3. Network（インフラネットワーク計算）

`computeNetwork()` — 毎 tick 開始時に BFS でネットワーク接続状態を再計算し、
各セルの `powered` / `watered` フラグを更新する。

### 4-4. Simulation（シミュレーション tick）

`tick()` — 3秒ごとに呼ばれる。以下の順で処理する。
1. `computeNetwork()` — インフラ接続判定
2. 収入・維持費計算
3. 人口増減計算
4. 犯罪・幸福度計算
5. イベント発生確率チェック
6. 年次更新
7. ゲームオーバー/クリア判定
8. `updateUI()` 呼び出し

### 4-5. Vehicle（乗り物アニメーション）

`spawnVehicles()` — tick ごとに道路・線路・空港の状態を確認し、適切な乗り物を生成。
`updateVehicles()` — 毎フレーム呼ばれ、乗り物の位置を更新。経路終端で消滅または折り返し。

### 4-6. Events（イベントシステム）

`processEvents()` — tick ごとに確率判定。イベント発生時に `events[]` キューへ追加。
`updateEvents()` — 毎フレーム呼ばれ、火災エフェクト等のアニメーションを更新。

### 4-7. Renderer（描画）

`draw()` — 毎フレーム。グリッド→乗り物→エフェクト→パーティクル→ホバーの順で描画。
`drawTile()` — セルタイプと状態（通電・断水・火災・ダメージ）に応じた描画を分岐。
`drawVehicle()` — 乗り物タイプ別の描画。
`drawInfoPopup()` — ホバー中セルの詳細ポップアップを Canvas 上に重ねて描画。

### 4-8. UI（HTMLイベントハンドラ）

`initToolbar()` — タブ切り替えと各ボタンのイベント登録。
`updateUI()` — ステータスバーの数値を DOM へ反映。

## 5. 交通システムの動作フロー

```
[住宅タイル配置]
     ↓
tick() で道路隣接チェック
     ↓
道路が連続する経路を BFS で探索
     ↓
spawnVehicles() で Car オブジェクト生成
path = [道路タイルの座標列]
     ↓
updateVehicles() で毎フレーム path 上を前進
progress += speed / TS  (progress: 0.0〜1.0)
     ↓
次タイルへ進む or 経路終端で折り返し
```

```
[駅×2タイル + 線路接続]
     ↓
tick() で駅間の線路経路を BFS 探索
     ↓
spawnVehicles() で Train オブジェクト生成
     ↓
updateVehicles() で線路タイル上を往復
```

```
[空港タイル配置]
     ↓
tick() でランダム間隔
     ↓
spawnVehicles() で Plane オブジェクト生成
x=-40, y=ランダム, dx=2, dy=-0.3 等
     ↓
canvas 右端を超えたら active=false で消滅
```

## 6. インフラ接続フロー（電力システム例）

```
発電所タイル[r][c]
  ↓ BFS
隣接する送電線タイルを繋がりとしてキューに積む
  ↓
送電線が接続する施設タイルに powered=true を付与
  ↓
tick() で powered=false の施設は収入・人口効果を計上しない
```

## 7. 災害・イベントフロー

```
tick() 内 processEvents() {
  // 火災
  if (random() < fireChance) {
    候補タイル（RES/COM）をランダム選択
    events.push({ type:'fire', r, c, timer:600 })
    grid[r][c].hp -= 50
  }
  // 地震
  if (year % 10 === 0 && random() < 0.3) {
    広域ダメージ: 全タイルに hp -= random(10,30)
    道路タイルは一部を破損(ROAD_BROKEN)に変更
    events.push({ type:'quake', severity:2, timer:180 })
  }
}

// 消防署が隣接する火災タイルは timer が速く減少（鎮火）
// 警察署は crimeRate を -20% 低下させる
```
