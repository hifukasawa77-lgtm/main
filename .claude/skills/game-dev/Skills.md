# ゲーム開発スキル集

Canvas API のみで構築するトップビュー / パズル系ゲームの実装ガイドライン。

---

## Skill: 衝突判定 (Collision Detection)
- **概要**: Canvas ゲームで多用する衝突判定パターンをケース別に使い分ける。
- **実装要件**:
  - **矩形同士 (AABB)**: `x1 < x2+w2 && x1+w1 > x2 && y1 < y2+h2 && y1+h1 > y2` で判定。最も軽量なので壁・床・アイテムに優先使用すること。
  - **円同士**: 2点間の距離 `Math.hypot(dx, dy)` が半径の和未満なら衝突。キャラクター同士や弾丸の判定に使用すること。
  - **矩形×円**: 矩形上の最近接点を求め、円の中心との距離で判定する。複合形状のヒットボックスに使用すること。
  - 判定頻度が高い場合は空間分割（グリッド / クアッドツリー）を導入し、総当たりを避けること。

## Skill: キャラクターAIパターン (Character AI Patterns)
- **概要**: 敵キャラクターやNPCの動作を状態機械で管理する。
- **実装要件**:
  - AIの状態は `{ state: 'idle' | 'patrol' | 'chase' | 'attack' | 'dead' }` のように文字列で明示的に管理すること。
  - 各状態の遷移条件をコメントで明記し、`update()` 内で `switch(this.state)` を使って処理を分岐すること。
  - **パトロール**: ウェイポイント配列を順に移動。端に達したら折り返すか循環させる。
  - **追跡**: プレイヤーとの角度を `Math.atan2(dy, dx)` で算出し、その方向へ一定速度で移動する。
  - **視野判定**: プレイヤーとの距離＋角度（内積）で扇形の視野範囲内かどうかを判定すること。

## Skill: ゲームループ設計 (Game Loop Design)
- **概要**: 入力 → 更新 → 描画のサイクルを安定して60fps で動作させる。
- **実装要件**:
  - デルタタイム (`deltaTime = (timestamp - lastTimestamp) / 1000`) を毎フレーム計算し、移動量を `speed * deltaTime` で算出することでフレームレート非依存にすること。
  - ループは `update(deltaTime)` と `draw()` を明確に分離し、描画ロジックに状態変更を混入させないこと。
  - `deltaTime` の上限を `Math.min(deltaTime, 0.1)` でキャップし、タブ非アクティブ復帰時の大ジャンプを防ぐこと。
  - ゲームの一時停止は `cancelAnimationFrame` ではなくフラグ (`isPaused`) で制御し、ループ自体は維持すること。

## Skill: Canvas API テクニック拡充 (Advanced Canvas Techniques)
- **概要**: 既存の有機的描画・グラデーション・エフェクトスキルを補完する高度なテクニック。
- **実装要件**:
  - **スプライトシート描画**: `drawImage(img, sx, sy, sw, sh, dx, dy, dw, dh)` の9引数形式でアニメーションフレームを切り出すこと。
  - **ピクセル操作**: `getImageData` / `putImageData` は重いため、エフェクト用途に限定し毎フレーム呼び出しを禁止する。
  - **クリッピング**: `ctx.clip()` でキャラクターの描画範囲をマスクし、ウィンドウ外へのはみ出しを防ぐこと。
  - **文字描画**: HUDのテキストは `ctx.textBaseline = 'top'` を統一設定し、`measureText` で幅を取得してセンタリングすること。
