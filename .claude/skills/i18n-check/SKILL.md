# /i18n-check — 日英バイリンガル一貫性

## 使い方

```bash
bash .claude/skills/i18n-check/i18n-check.sh            # 全HTML
bash .claude/skills/i18n-check/i18n-check.sh zelda_like.html  # 指定ファイル
```

終了コード: 必須項目OK=0 / 問題あり=1。検査項目:
- `<html lang="...">` の有無（必須）
- `data-i18n` 属性を1つも持たないポータル系ページ（警告）
- `<title>` が日本語のみ/英語のみ（警告。日英併記が既存慣例）

## 本サイトの日英表記ルール（index.html から抽出した慣例）

1. **UIテキストは `data-i18n` 属性で切替**（index.html の言語切替機構）。ポータル系ページの新規UI文言は必ず `data-i18n` キーを付与し、日英両方の辞書エントリを追加する
2. **`<title>` / OGP は「日本語名 | 英語名」併記**（例: `将棋パズル | Shogi Puzzle`）
3. **ゲーム内テキストは英語ベース＋起動画面/説明欄に日本語**（Canvas内の動的文言まで切替機構を持ち込まない。操作説明はページ側のDOMに日英併記）
4. **`lang` 属性**: 全ページ `<html lang="ja">`。英語のみのページを作る場合は `lang="en"`
5. 数値・日付のロケール表記は `toLocaleString('ja-JP')` / `('en-US')` を言語設定に連動させる（ポータル系のみ）

## 是正手順
- `lang` 欠落 → `<html>` タグに追記（1行・安全）
- タイトル片言語 → 「日本語名 | English Name」に統一
- 新規辞書エントリ追加時は既存キーの命名規則（`セクション-要素` 形式、例: `tool-link-play`）に合わせる
