# /weekly-review — 週次振り返り

日々の記録（/second-brain）と都度の昇格（/self-improve）の間を埋める、週単位の定期レビュー。**self-improve が「個別の学びを狭く昇格させる」**のに対し、**weekly-review は「1週間を俯瞰して傾向を見つける」**（同種ミスの再発・停滞タスク・昇格漏れの発見が目的）。

## 手順

### 1. 素材収集
```bash
bash .claude/skills/weekly-review/gather.sh     # 直近7日のDaily/ADR/コミットを一括表示
```

### 2. 俯瞰して3分類に整理
素材を読み、以下を各2〜5行でまとめる:
- **Keep（続けること）**: うまく機能した進め方・ルール
- **Problem（改善すること）**: 停滞・手戻り・同種ミスの再発（→ 再発は /self-improve の最優先昇格候補）
- **Try（来週やること）**: 具体的な次アクション（持ち越しタスクの棚卸しを含む）

### 3. 週次ノートをVaultへ書く
`obsidian-vault/01-Daily/YYYY-MM-DD.md`（実施日）に「## 週次振り返り（MM/DD〜MM/DD）」セクションとして追記する。Daily圧縮ルール準拠（要点のみ・詳細はADR/知見へのwikilink）。

### 4. 昇格候補があれば /self-improve を起動
Problem に「ルール化されていない再発」があれば、その場で /self-improve を実行して昇格まで済ませる（振り返りっぱなしにしない）。

## 原則
- 週次ノートは**次の行動につながる形**で書く（感想ではなくKeep/Problem/Try）
- 持ち越しが2週連続したタスクは「やる/やらない/分割する」を深澤に確認する
