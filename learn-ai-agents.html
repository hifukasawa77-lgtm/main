🤖 AIコーディングエージェント

ターミナルやIDE上で動作し、コードの読み書き・コマンド実行・テストなどを自律的に行う「AIコーディングエージェント」。Claude Code・Codex・Gemini CLIなど主要なエージェントの種類と、それらを動かす「ハーネス（実行基盤）」の仕組みを解説する。

📌 主要AIコーディングエージェント比較

エージェント開発元実行環境特徴
Claude Code Anthropic ターミナル(CLI)、VS Code等のIDE拡張、SDK、Web版 エージェントループ＋ツール呼び出し（Bash・Read・Edit・Grep等）で自律的にタスクを遂行。サブエージェントやMCP連携、CLAUDE.mdによるプロジェクト固有指示、ツール実行ごとの権限確認モデルが特徴。
Codex / Codex CLI OpenAI ターミナルCLI、クラウド型エージェント、IDE拡張 コーディングに特化したモデルをベースに、サンドボックス内でコード実行・テストを行う。複数タスクをクラウド上で並列実行するエージェントモードを持つ。
Gemini CLI Google ターミナルCLI（オープンソース） Geminiモデルをベースにした無料利用枠の大きいCLIエージェント。検索やファイル操作などのツールと連携し、MCPにも対応。
GitHub Copilot (Agent mode) GitHub / Microsoft VS Code、GitHub.com Claude・GPT・Geminiなど複数モデルを選択可能。Issueの自動対応やPRレビューなど、リポジトリに統合された自律エージェント機能を提供。
Cursor Anysphere AIネイティブIDE（VS Codeベース） Composer/Agentモードでマルチファイル編集を自動実行。複数モデルの切替やコードベース全体のインデックス検索が可能。

いずれも「LLM本体」＋「LLMにツールを使わせて結果を観察させる仕組み（ハーネス）」という構成は共通しており、対応モデルや実行環境、安全機構（権限確認・サンドボックス）に違いがある。

📌 エージェントの動作の仕組み（エージェントループ）

AIコーディングエージェントは、以下のループを繰り返しながらタスクを完了させる。

1. ユーザーがタスクを入力する（例: 「バグを修正して」）
        ↓
2. LLMがタスクを分析し、必要なツールと引数を決定する
   （例: Read("src/app.js"), Grep("function foo")）
        ↓
3. ハーネスがツールを実際に実行し、結果（ファイル内容・
   コマンド出力・エラー等）をLLMに返す
        ↓
4. LLMは結果を踏まえて次のアクションを決定する
   （さらにツールを呼ぶ／ユーザーに確認する／完了報告する）
        ↓
タスク完了までステップ2〜4を繰り返す（エージェントループ）

このループにより、エージェントは「一度の応答で全てを書き切る」のではなく、コードを読んで状況を確認しながら段階的に修正・検証を進められる。

📌 主要概念・用語

用語解説
ハーネス (Harness) エージェントを動かすための周辺システム全体。CLI、権限管理、コンテキスト管理、ツールの実装などを含む「モデルを動かす土台」を指す。同じLLMでもハーネスが異なれば挙動や安全性が大きく変わる。
ツールユース / Function Calling LLMが「この関数（ツール）をこの引数で呼びたい」と構造化データで出力し、ハーネス側がそれを実行して結果をLLMに返す仕組み。ファイル操作・コマンド実行・Web検索などをLLMが間接的に行うための基盤。
MCP (Model Context Protocol) LLMアプリケーションと外部ツール・データソースを接続するための標準プロトコル。Anthropicが提唱し、各社のエージェントが対応を進めている「USB-Cポート」のような共通インターフェース。
サブエージェント (Subagent) メインのエージェントが特定のタスクを専用の子エージェントに委任する仕組み。並列処理や、メインのコンテキストを圧迫しないための調査タスクの切り出しなどに使われる。
コンテキストウィンドウ LLMが一度に処理できる入力＋出力の上限トークン数。長い会話やコードベースを扱う際は、この範囲に収まるように情報を要約・取捨選択する必要がある。
システムプロンプト / プロジェクト指示ファイル エージェントの振る舞いを定義する初期指示。CLAUDE.mdやAGENTS.mdのように、プロジェクト固有のルール・コーディング規約をリポジトリに置いて読み込ませる手法が広まっている。
サンドボックス実行 コード実行やファイル操作を隔離された環境で行い、ホストシステムや本番環境への意図しない影響を防ぐ仕組み。
権限モード (Permission Mode) ツール実行（ファイル編集・コマンド実行など）の前にユーザーへ確認を求めるか、一定範囲で自動実行を許可するかを制御する設定。安全性と作業効率のバランスを取るための仕組み。
